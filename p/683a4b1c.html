将个人博客免费托管在 Netlify 上
Zhicheng Lv3
  2021-11-07 18:18:58 2021-11-07 18:18  
  • 博客相关

本站目前使用的是 Hexo + GitHub Pages 搭建技术，由基于 Node.js 的静态博客框架 Hexo 渲染生成静态网页，之后将存放着目标文件的 public 文件夹自动同步至 GitHub 仓库中。

说点儿题外话，我写博文的正常流程是 Terminal 中打开博客所在文件夹，hexo new "文章标题" 先新建一个 markdown 文档（这样会自动生成文章标题下方的元信息），填写相关的标签以及分类、头图、摘要等信息之后，就可以正式进入码字的阶段了。之后回过头来再补充或者修改当然也是可以的。

写完全文，配上图片之后，一篇待发布的文章就弄好了。这时候回到 Terminal 输入 hexo clean 以清除缓存的网页文件, hexo g 以生成新的静态网页文件，hexo s 启动本地预览, 如果生成的网站无误就可以退出本地预览状态，输入 hexo d 将博客上传至 GitHub 中相对应的仓库，之后 GitHub Pages 就会根据最新的仓库内容更新网站了。

这一番操作下来其实不如想象中的优雅，但对于我个人来说习惯了之后并不繁琐，暂时还可以接受以这种流程来进行写作。毕竟，坚持写下去、发出来才是最重要的，未来要是有更加高效简练的方式，完全可以到时再来学习。

不过，网站部署在 GitHub 上访问速度不稳定又是一个不争的事实，虽然说这就是免费产品的代价吧。刚好看到其他博主有将博客同时部署在 GitHub Pages 和 Netlify 上的操作，并且部署过程也相对简单，这就值得一试了。

Step 1：注册 Netlify

我使用的是直接用 GitHub 账号创建 Netlify 账号，因为我的目标就是直接将 GitHub 仓库（也就是博客的托管仓库）的代码部署到 Netlify。

Step 2：创建站点

进入个人页面后点击 New site from Git,之后会给你三个部署来源的选择（GitHub、GitLab、Bitbuckt），毫无疑问我选择 GitHub，然后按照提示授权、选仓库，最后点击 Deploy site 开始部署。

Step 3：自定义域名

部署需要几分钟的时间，完成之后 Netlify 会自动分配一个站点域名给你，可以通过站点设置选项进行更改，但这时候它最多能够提供给你的域名也就是 自定义名称.netlify.com 这样，然后就需要设置自己的域名了。
