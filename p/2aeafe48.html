Hexo Fluid主题轻松启用Waline评论系统

即使是无人问津的博客，文章下方没有评论系统也是不太像话的，于是今天的任务就是为个人网站开启评论系统。

评论系统的选用

原本我是不知道评论系统还有那么多门门道道的，直到看了一张各种评论系统的比较及选择，其中讲到 valine 时说它“免认证，国内高速，免登录”，而且看起来是一个相当简洁的布局。

不过，在看一篇介绍如何安装 valine 的文章时，发现该博客底下的评论系统 Powered by Waline v1.3.10，Waline 又是什么？

“他在介绍 valine 自己却用了这个 waline，是不是后者更好用呢？”我是这么想的。最后临时决定换成 waline 了。

安装说明

开启过程比我想象中的要简单得多，全程都是根据Waline官方文档的指导完成了部署，并且在准备将其引入至个人网站时发现，Fluid 主题里已经内置了相应的代码，只需要在主题配置文件中开启并指定就可以了。

具体步骤

先打开 Waline 快速上手指南，完成数据库设置和服务端设置，接着根据 Fluid 主题配置指南在配置文件中开启 Waline 评论功能，最后注册管理端账号。

LeanCloud 设置 (数据库)

1、登录或注册 LeanCloud 国际版并进入【控制台】；
2、点击【创建应用】后填写应用名称，计价方案选择开发版然后点击【创建】即可；
3、进入应用，选择左下角的【设置】 > 【应用凭证】。你可以看到你的 APP ID,APP Key 和 Master Key。这一步的目的就是获取这三个参数。

Vercel 部署 (服务端)

1、点击这个链接，跳转至 Vercel 进行 Server 端部署；

2、如果你未登录的话，Vercel 会让你注册或登录，请使用 GitHub 账户进行快捷登录；

3、输入一个你喜欢的 Vercel 项目名称并点击 Create 继续；

4、此时会让你创建 Team 账号，这里选择【skip】即可；

5、此时 Vercel 会基于 Waline 模板帮助你新建并初始化仓库，仓库名为你之前输入的项目名；

一两分钟后，满屏的烟花会庆祝你部署成功。此时点击 【Go to Dashboard】 可以跳转到应用的控制台。

6、点击顶部的 【Settings - Environment Variables】 进入环境变量配置页，并配置三个环境变量LEAN_ID, LEAN_KEY 和 LEAN_MASTER_KEY 。它们的值分别对应上一步在 LeanCloud 中获得的 APP ID, APP KEY, Master Key。

7、环境变量配置完成之后点击顶部的 【Deployments】 点击顶部最新的一次部署右侧的 【Redeploy】 按钮进行重新部署。该步骤是为了让刚才设置的环境变量生效。

8、此时会跳转到 【Overview】 界面开始部署，等待片刻后 STATUS 会变成 Ready。此时请点击 【Visit】 ，即可跳转到部署好的网站地址，此地址即为你的服务端地址。

待会儿配置的时候需要用到服务端地址，即serverURL。

HTML 引入 (客户端)

这时候可以打开Fluid 主题配置指南，根据指引修改 Fluid 主题配置文件。

1、在主题配置中开启并指定评论模块：

开启评论模块，并指定插件为 waline。

2、配置 waline 参数

这一步比较简单，将上一步获取到的服务端地址填入serverURL中即可。

保存文件后网站的评论服务就已经成功开启了。命令行中输入hexo clean,hexo g,接着hexo s开启本地服务器预览，之后就能在文章页的下方看到评论区块了。

评论管理 (管理端)

1、部署完成后，访问 <serverURL>/ui/register 进行注册。首个注册的人会被设定成管理员;

2、管理员登陆后，即可看到评论管理界面。在这里可以修改、标记或删除评论;

3、用户也通过评论框注册账号，登陆后会跳转到自己的档案页。

总结

在安装过程中，我按照以上两个教程一步步顺下来，几乎没碰到什么困难。之后经过测试，评论功能也是完全可以使用的，就是用户评论会直接放出来而不是进入审核环节，管理员审核完成后才放出。

可能有需要重新设置的地方，不过我暂时不想弄了，毕竟短期内不会有多少留言。
