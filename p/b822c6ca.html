本文为 kaggle 平台数据清洗（Data Cleaning）课程第一课讲解部分的编译内容。

课程总共由五部分组成，即处理缺失值（Handling Missing Values）、数据归一化与标准化（Scaling and Normalization）、解析数据（Parsing Dates）、字符编码（Character Encodings）、数据输入不一致（Inconsistent Data Entry）。

课程旨在帮助学员掌握高效的工作流程，清理现实工作中的混乱数据。导师为 Rachael Tatman。

预备知识：Pandas

Pandas 是一个开源的，BSD 许可的库，为 Python 编程语言提供高性能，易于使用的数据结构和数据分析工具。

Python 在数据处理和准备方面一直做得很好，但在数据分析和建模方面就没那么好了。Pandas 帮助填补了这一空白，使用户能够在 Python 中执行整个数据分析工作流程，而不必切换到更特定于领域的语言，如 R。

Step 3: 分析数据缺失的原因

到此我们真正地进入了“数据科学”的部分，这意味着：深入观察你的数据，并试图弄清楚它为什么是这样的，以及这将如何影响你的分析。

这可能是数据科学中令人不知如何下手的部分，特别是如果你还只是这个领域里没有多少数据分析经验的新手。

对于处理缺失值，我们有时候需要用直觉来找出数值缺失的原因。你可以问自己一个关键问题，以帮助弄清这个问题：

这个数值丢失是因为没有记录，还是因为它本身就不存在？

如果一个数值是因为本身不存在而缺失（比如记录一个还没有生育的夫妇，他们孩子的身高是多少），那么试图猜测它可能是什么就没有多大意义了，这样的数值记录为NaN也就顺理成章了。

另外一个情况，如果一个数值是因为没有记录而丢失，那么你可以根据该列和该行的其他值来预测它的数值可能是什么。这就是所谓的归纳法，我们接下来将学习如何进行归纳。

通过查看原始数据的文档，我们了解到“TimesSec”这一列代表的是回合结束时，比赛时间还剩多少秒。也就意味着，数值缺失不是因为它不存在，而是因为没有记录到。因此，我们有必要尝试猜测它们的数值应该是什么，而不是将其当做NA。

另外，有一些列比如说“PenalizedTeam”也存在着缺失值，而这一列代表的是被罚的球队，如果场上没有出现判罚情况的话，这一栏出现缺失值是正常的。不过，如果将其置空或者添加表示对阵双方都没有被罚情况的“neither”以代替缺失值会让数据看起来更容易理解。

小贴士：阅读数据集的说明文档可以让你对所要分析的数据理解得更为透彻。如果数据集来自于第三方，那就尝试联系他们以获得更多信息吧。

如果你所做的数据分析项目需要相当的精确度，那么你需要逐一查看每一列，找出填补那些缺失值的最佳处理方式。

在本课程中，我们会介绍一些简单快速的手段来处理缺失值，但请记住这样的方法也会在无意中删除一些有用的信息，或者在数据中添加一些干扰因素。

Step 4: 删除缺失值
